Layout

Reusable UI Components 

These live in view/module. The order and placement of the following components are important due to dangling HTML tags... Check out view/index.html for details.

head.html

The <head> block. Stylesheets, <meta> tags, and the title of the page go here.

Do not place <script> tags with src attributes in the <head> block. Stylesheets are going to slow down your page loads badly enough...

navbar.html  The pretty black bar up there. Check out the docs for details pertaining to Bootstrap's Navbar formatting.
foot.html

 That magic grey bar that's always at the bottom of the page. Have a look at @mdo and @fat's example to see how it's done.

<script> tags with src attributes do go here! While this is a questionable practice according to the W3C standard, placing your script tags at the end of the body will allow your page to render without waiting for jQuery and the Bootstrap pluggins to download in serial. This also means that you can't put on{SomeEvent} attributes or other <script> tags that depend on these scripts in your main HTML. Instead, put all of your scripts after their dependencies at the end of the <body> and use JQuery to register event handlers.

Flair

Glyphicons

In addition to the Glyphicons Halflings icon-set included in Bootstrap, the Glyphicons Free icon-set has been packaged in resource/icon/glyphicons with a stylesheet that allows icons to be used with the same syntax as Halflings from bootstrap.

How to use

All icons require an <i> tag with a unique class, prefixed with glyph-, instead of icon-. 

<i class="glyph-search"></i>

Google Prettify

The minified source for google-code-prettify from the Bootstrap documentation site is included for your stylistic convenience.

How to use

Wrap codeblocks in <pre> tags with the prettyprint class, and optionally, the linenums class to add line numbers. Be careful to remove leading whitespace inside the block. Prettify will render exactly what it gets, indents and all! 

<pre class="prettyprint linenums">
	<i class="glyph-search"></i>
</pre>
